Импорт базы данных MySQL может осуществляться несколькими способами. Один из самых популярных методов — использование командной строки через утилиту `mysql`. Вот пошаговая инструкция:

### Шаг 1: Подготовка

Убедитесь, что у вас есть файл дампа базы данных в формате `.sql` или `.sql.gz`, который вы хотите импортировать. Этот файл обычно создается при экспорте базы данных командой `mysqldump`.

### Шаг 2: Импорт базы данных

#### Вариант 1: Импорт в существующую базу данных

Если база данных уже создана, вы можете использовать следующую команду для импорта:

```bash
mysql -u username -p database_name < dump_file.sql
```

Где:
- `username` — имя пользователя MySQL, имеющего доступ к базе данных.
- `database_name` — название базы данных, куда будет производиться импорт.
- `dump_file.sql` — путь к файлу дампа базы данных.

После ввода команды вам нужно будет ввести пароль пользователя MySQL.

#### Вариант 2: Создание новой базы данных перед импортом

Если база данных еще не существует, сначала создайте её, а затем выполните импорт:

1. Создаем новую базу данных:
   ```bash
   mysql -u root -p -e "CREATE DATABASE new_database;"
   ```
   
   Здесь `root` — пользователь с правами администратора, а `new_database` — название создаваемой базы данных.

2. Выполняем импорт в созданную базу данных:
   ```bash
   mysql -u username -p new_database < dump_file.sql
   ```

### Шаг 3: Проверка

Чтобы убедиться, что импорт прошел успешно, подключитесь к базе данных и проверьте наличие таблиц и данных:

```bash
mysql -u username -p new_database
```

Внутри консоли MySQL введите:

```sql
SHOW TABLES;
SELECT * FROM table_name LIMIT 10;
```

Это позволит увидеть список таблиц и данные в одной из них.

### Дополнительная информация

#### Если файл дампа заархивирован (.gz)

Если ваш файл дампа находится в архиве формата `.gz`, то его можно распаковать прямо во время импорта:

```bash
gunzip -c dump_file.sql.gz | mysql -u username -p database_name
```

Здесь команда `gunzip -c` распаковывает файл и передает его содержимое непосредственно в `mysql`.
