title: Как установить Windows Subsystem for Linux (WSL) от имени администратора без создания нового пользователя

---

Windows Subsystem for Linux (WSL) – это инструмент, который позволяет запускать различные дистрибутивы Linux прямо внутри операционной системы Windows. По умолчанию при установке WSL создается новый пользователь для работы в среде Linux, но иногда может потребоваться установить WSL от имени администратора без создания этого пользователя. В этой статье мы рассмотрим пошаговую процедуру установки WSL таким образом.

#### Шаг 1: Проверка наличия WSL

Прежде чем приступить к установке, убедитесь, что у вас уже установлен компонент WSL. Для этого откройте командную строку от имени администратора и выполните следующую команду:

```bash
wsl --list --verbose
```

Эта команда покажет список установленных дистрибутивов WSL и их состояние. Если вы видите сообщение о том, что WSL не установлен, перейдите к следующему шагу.

#### Шаг 2: Установка WSL через PowerShell

Для установки WSL выполните следующие шаги:

1. Откройте PowerShell от имени администратора.
2. Выполните следующую команду для включения компонента WSL:

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
```

3. После выполнения команды перезагрузите компьютер.

#### Шаг 3: Загрузка дистрибутива Linux

После того как WSL был включен, вам нужно выбрать и загрузить нужный дистрибутив Linux. Вы можете найти доступные дистрибутивы в магазине приложений Microsoft Store. Например, чтобы установить Ubuntu, выполните следующие шаги:

1. Перейдите в магазин приложений Microsoft Store.
2. Найдите и установите дистрибутив Ubuntu.

#### Шаг 4: Запуск дистрибутива без создания пользователя

По умолчанию после первой загрузки дистрибутива будет предложено создать нового пользователя. Чтобы избежать этого шага, выполните следующие действия:

1. Откройте командную строку от имени администратора.
2. Запустите дистрибутив командой:

```bash
ubuntu.exe config --default-user root
```

Это позволит вам войти в систему под пользователем `root` сразу же после запуска дистрибутива.

#### Шаг 5: Настройка окружения

Теперь, когда вы вошли в систему под пользователем `root`, можно настроить окружение по своему усмотрению. Например, вы можете установить необходимые пакеты и программы, используя менеджер пакетов вашего дистрибутива. Для Ubuntu это будет APT:

```bash
apt update && apt upgrade
```

Этот шаг обновит списки пакетов и установит последние обновления.

#### Заключение

Таким образом, установка WSL от имени администратора без создания нового пользователя возможна и требует всего нескольких шагов. Это может быть полезно в ситуациях, когда требуется полный контроль над системой или необходимо работать исключительно от имени суперпользователя. Однако следует помнить, что работа под пользователем `root` несет определенные риски безопасности, поэтому используйте этот метод с осторожностью.
